  1. Que es un sistema de control de versiones?

    Un sistema de control de versiones es una herramienta de softwware que nos permite monitorizar y gestionar cambios en un sistema de archivos. Son ejemplos de sistemas de control de versiones Git, CVS o ClearCase

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones mientras que github es un sitio web que proporciona un servidor Git con una serie de herramientas extra para trabajar con repositorios git.

  3. Cómo creo una rama en git?

    Para crear una rama en git ejecutamos el comando git branch [nombre] donde nombre es el nombre de la rama a crear

  4. Cómo clono un repositorio?

    Para clonar un repositorio utilizo el comando git clone [enlace] donde enlace es el enlace al repositorio

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Para bajar o actualizar el Repositoio Local utilizamos el comando git pull [alias] [rama]

  6. Cómo puedo tener un tag?

    Para tener un tag utilizo el comando git tag

  7. Cómo hago una mezcla de ramas?

    Para hacer una mezcla de ramas utilizamos el comando git merge

  8. Cómo puedo moverme a un commit especifico?

    Utilizamos el comando git log para ver los números correspondientes a los commits y luego utilizar el comando git checkout [númeroCommit]. Si el commit tiene un tag asignado, podremos utilizar el comando git checkout [tag]

  9. Cómo puedo borrar una rama?

    Con el comando gti branch -d [nombre]

  10. Cómo puedo borrar un tag?

    Con el comando git tag -d [nombre]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comando git status
